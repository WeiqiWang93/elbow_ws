Installation

From source code

Just download or update from git:

  • git clone https://github.com/offworld-projects/swarm-robotic-mining/

COTS install

Only needed for the first install. The access to Internet is required.

  • cd <bot_overseer path>/scripts/install/
  • ./install_cots

Deploy data

Some data including database and default missions definitions are required to be deployed in the expected target folder.

  • If not present, create a folder /home/offworld/ with rw permissions for the user to execute the software.
  • cd <bot_overseer path>/scripts/install/
  • ./deploy_data

Start-up

ROS core

Make sure ROS core is running

Start overseer

  • cd <bot_overseer path>/scripts/install/
  • ./overseer --model <bot model>

Supported bot models are the following:

  • CRUSHER
  • DIGGER

The overseer also supports a test mode. In this mode test APIs and test ROS actions are started as wll as a turtlebot navigation stack.

  • ./overseer --model <bot model> --testmode

Missions

Definitions

Mission definitions are located at /home/offworld/data/missions A mission is an ASCII file which defines the operations of the given mission. One operation is specified per line as per the example below:

op(CRUSHER_ON)

op(PAUSE_MISSION)

op(CRUSHER_OFF)

The available operations are specified in the database located at /home/offworld/data/database, in the Operations tab. The arguments associated with an operation are specified in the Arguments tab

Asynchronous operations

Several asynchronous operations that take time to complete can run in parallel. For this purpose set the flag wait_ok to False:

op("LONG_OP", wait_ok=False)

Waiting

A mission can wait for a delay or some previous operation(s) to complete. The WAIT_MISSION operation provides the following arguments:

  • delay: a delay in seconds to wait for.
  • wait_ops: a flag that indicates whether to wait for previous operations to complete.
  • ops: specifies the list of operation indexes to wait for. When not given, the wait operation waits for all the previous operations to be completed.

op("WAIT_MISSION", wait_ops=True, ops=[7])

Scripting

A mission can also include one or several python scripts either:

  • At the main scope of the mission: the script is executed upon loading the mission
  • Within a specific SCRIPT_MISSION operation scope: the script is executed upon execution of the operation. In particular this allows to update the mission automatically based on results from previous operations of the same mission.

Control from shell

The operator can control the mission operations from command line. For this purpose:

  • cd <bot overseer="" path>="">/scripts/shell

You can then control the mission with the following commands:

  • Start the mission: ./mission start
  • Pause the mission: ./mission pause
  • Resume the mission: ./mission resume
  • Stop the mission: ./mission stop
  • Load a mission: ./mission load "mission=<mission name>". The mission loaded at start-up is called default
  • Execute a "manual" operation on-demand (only possible when the mission is paused): ./mission run_op <op name> <args>
  • Insert an operation on-demand (only possible when the mission is paused): ./mission insert_op <op name> <index> <args>
  • Jump to a specific operation (only possible when the mission is paused): ./mission jump_to_op index=<index>
  • Help of the command line: ./mission help
