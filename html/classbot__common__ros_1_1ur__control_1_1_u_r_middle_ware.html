Detailed Description

Class to handle interfacing to the UR10 libraries and provide high-level functionality.
This handles all the parameters needed to run the robot, which can be provided from any source, including yaml files.

Moves the arm with or without motion planning (determined by input arg 'use_mp' ). Movement is done either via cartesian poses
    or joint states. With motion planning, movements are always through joint states. 
    Arg target_points is always a list of either cartesian poses or joint states. Here, a joint state in turn should be a list of joint
    values and a cartesian pose should be an object of  m3d.Transform
    The Velocity and the acceleration will be interpreted as rad/s(2) for type = js and m/s(2) for type = ls.
    Target type can only take values 'js or 'ls'.
