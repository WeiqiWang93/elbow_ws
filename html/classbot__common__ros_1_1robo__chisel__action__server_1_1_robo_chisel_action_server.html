def bot_common_ros.robo_chisel_action_server.RoboChiselActionServer.checkAngleLimits_NEW (   self,
  pose,
  maxAngle,
  dir = 'y' 
) 
use quaternions to find required rotations in tool frame which are in direction of 'maxAngle' but valid w.r.t. max-allowable-angles

refs:
https://www.gamedev.net/forums/topic/646852-enforce-angle-limits-on-a-quaternion/
http://www.euclideanspace.com/maths/geometry/rotations/for/decomposition/

q = ( x, y, z, w ). You decompose into q = q_xy * q_z where q_z = ( 0, 0, z, w ) / sqrt( z^2 + w^2 ) 
and q_xy = q * conjugate( q_z ). Then you apply the limit and multiply by q_xy.

use motion planner black-box collision check to find a max allowable yaw
given the current octomap using a binary search method.

Recall, euler angles are order YXZ
Input args:
